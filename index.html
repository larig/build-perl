Build a custom perl

Stock Binary

If you just want a perl binary, are flexible about which version, not fussed about whether it is threaded, or optimised for your architecture, then you most likely want a stock perl, which will be waiting for download from wherever you normally get your packages. (If you are on Windows then Strawberry Perl is a good option and the options below are most likely not open to you without something like cygwin.) One advantage of this route is you are using an identical binary to many similar users.

Perlbrew

If you want a specific version of perl, and it's not important where it goes, nor what library paths it uses, nor what environment variables are needed, then a lot of people recommend perlbrew. Some advantages of this route are that it is fast to switch between perl versions and many perl developers use the same approach.

Standard Build

If you want a specific version of perl, optimised for your hardware, possibly choosing its installation path, with minimal environment variables and the option of later customising the library path, then use the standard build at Get Perl 

$ ./Configure -des -Dprefix=/opt/perl
$ make && make test && make install
and come back here if you want further customisations.

Custom Build

If you are wanting a perl v5.12 or above then there is a very easy route to customising where everything goes, including the library paths that are searched by the binary. The steps are just a small extension to the Standard Build instructions. 

$ ./Configure -des -Dprefix=/opt/perl
$ rm config.sh
$ vi Policy.sh
$ ./Configure -des -Dprefix=/opt/perl
$ make && make test && make install
There are some checks you can do along the way to see the effects before running the make step and the fuller story is at Build your own custom perl
